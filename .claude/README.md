# Universal Experience - Claude開発記録

このディレクトリには、Universal Experienceプロジェクトの構想、設計、進捗に関する詳細な記録が保存されています。

## 📁 ディレクトリ構成

```
.claude/
├── README.md                  # このファイル
├── project-vision.md          # プロジェクト構想
├── design-decisions.md        # 設計方針と技術的決定事項
└── progress.md                # 開発進捗状況
```

## 📄 ドキュメント概要

### 1. project-vision.md
**プロジェクト構想書**

Universal Experienceの理念、目標、ビジョンを記録したドキュメント。

**内容**:
- プロジェクトミッション
- 3つの「ユニバーサル」コンセプト
- 対象ユーザー
- 短期・中期・長期目標
- ブランディング戦略
- プラットフォーム戦略
- 差別化要因
- 成功指標（KPI）
- 5年後のビジョン
- パートナーシップ機会

**対象読者**: プロジェクト関係者全員、投資家、パートナー候補

### 2. design-decisions.md
**設計方針と技術的決定事項**

アーキテクチャ設計、技術選定、実装方針などの技術的決定を記録。

**内容**:
- アーキテクチャ設計（レイヤード構造）
- Flutter採用の理由
- 状態管理（Provider選定）
- UI/UX設計（Material Design 3）
- 色覚アルゴリズム設計（LMS変換）
- プラットフォーム別実装戦略
  - Android: AccessibilityService
  - Windows: Magnification API
  - macOS: CoreGraphics
  - Linux: Compositor
- プラグインアーキテクチャ
- セキュリティ設計
- パフォーマンス設計
- コーディング規約
- テスト戦略
- バージョニング戦略
- 国際化計画
- 技術的負債管理

**対象読者**: 開発者、コントリビューター、技術レビュアー

### 3. progress.md
**開発進捗状況**

日々の開発進捗、完了タスク、次のステップを追跡。

**内容**:
- 全体進捗（Phase別）
- 完了タスク（日付別）
- 進行中タスク
- 次のステップ（短期/中期/長期）
- コードメトリクス
- 既知の問題
- Phase完成条件
- 学んだこと
- マイルストーン

**対象読者**: 開発チーム、プロジェクトマネージャー

## 🎯 このディレクトリの目的

### 1. 透明性の確保
プロジェクトの意思決定プロセスを明確に記録し、誰でも参照できるようにします。

### 2. ナレッジの蓄積
技術的な選択理由、検討した代替案、将来の課題を文書化し、プロジェクトの知的資産として保存します。

### 3. オンボーディング支援
新しいコントリビューターがプロジェクトの全体像を素早く理解できるようにします。

### 4. 継続性の維持
開発が中断しても、ドキュメントを読めば誰でも再開できる状態を保ちます。

### 5. 意思決定の振り返り
過去の決定を振り返り、学びを次のフェーズに活かします。

## 📖 使用方法

### 開発者向け

```bash
# プロジェクトに参加する前に
1. project-vision.md を読んでプロジェクトの理念を理解
2. design-decisions.md で技術スタックと設計方針を確認
3. progress.md で現在の進捗状況を把握

# 開発中
- 新しい技術的決定 → design-decisions.md に追記
- タスク完了 → progress.md を更新
- 重要なマイルストーン → progress.md に記録
```

### プロジェクトマネージャー向け

```bash
# 定期レビュー（週次）
1. progress.md の進捗を確認
2. 完了タスクと次のステップを評価
3. リスクと技術的負債を確認

# 意思決定時
1. design-decisions.md で過去の決定を参照
2. 新しい決定を文書化
3. project-vision.md との整合性を確認
```

### コントリビューター向け

```bash
# 初回コントリビューション前
1. すべてのドキュメントを読む
2. プロジェクトの方向性を理解
3. 現在のフォーカスエリアを確認（progress.md）

# コントリビューション時
- 設計方針に従う（design-decisions.md）
- 進捗を更新（progress.md）
- 新しいアイデアは提案として記録
```

## 💡 ベストプラクティス
